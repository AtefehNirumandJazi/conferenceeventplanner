# ConferenceEventPlanner - Enhanced Version

A full-stack conference event planning application built on top of code generated by BESSER, then extended with custom features including event management, session scheduling, speaker assignment, and room allocation.


## Domain Model

The application was generated from the following B-UML class diagram, designed in the BESSER online editor:

![Class Diagram](Class%20Diagram.png)

The domain model includes:
- **Event** — name, startDate, endDate, location
- **Session** (abstract) — title, description, sessionType; belongs to an Event and a Room
- **Speaker** — fullName, affiliation, email; many-to-many with Session
- **Room** — name, capacity; one-to-many with Session
- **ScheduleSlot** — startTime, endTime; one-to-one with Session

## From Generated Scaffold to Full Application

This version demonstrates how BESSER-generated code serves as a solid foundation that can be extended with domain-specific features. The following capabilities are available:

### 1. Event Management
- Create, read, update, and delete conference events
- Track event dates and locations

### 2. Session Scheduling
- Manage sessions with titles, descriptions, and types
- Assign sessions to events, rooms, and schedule slots

### 3. Speaker Management
- Maintain speaker profiles (name, affiliation, email)
- Assign multiple speakers to sessions (many-to-many)

### 4. Room Allocation
- Define rooms with name and capacity
- Assign rooms to sessions

### 5. Schedule Slots
- Define time slots with start and end times
- Link schedule slots to sessions (one-to-one)

### 6. Dashboard & Charts
- Visual dashboard with Recharts-based analytics

## Getting Started

### With Docker (recommended)

```bash
docker-compose up --build
```

- Frontend: http://localhost:3000
- Backend API docs: http://localhost:8000/docs

### Without Docker

**Backend:**

```bash
cd backend
pip install -r requirements.txt
uvicorn main_api:app --host 0.0.0.0 --port 8000
```

**Frontend:**

```bash
cd frontend
npm install
npm run dev
```

## Optional Environment Variables
